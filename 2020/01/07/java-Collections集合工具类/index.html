2.注意

(1)sort(List list)使用前提：
  • 被排序的集合里面存储的元素，必须实现comparable，重写接口中的方法compareTo定义排序的规则
(2)Comparable接口的排序规则：
  • 自己(this)-参数 ：升序
  • 参数-自己(this)：降序
(3)Comparator和Comparable的区别
  • Comparable:自己(this)和别人(参数)比较，自己需要实现Comparable接口，重写比较的规则compareTo方法
  • Comparator:相当于找一个第三方的裁判，比较两个对象
(4)Comparator的排序规则：
  • o1-o2：生序
  • o2-o1：降序
